# **DexGen Project Bible**

**Distributed Generative Pipeline for Starsilk**

## **1\. Executive Summary**

DexGen is a distributed, zero-cost image and video generation pipeline designed to decouple high-compute inference from local orchestration.

* **Orchestrator:** MacBook Air (M1) running a Gradio-based UI.  
* **Render Node:** Google Colab (NVIDIA T4 GPU) running a headless FastAPI service.  
* **Objective:** Reliable generation of Starsilk assets with strict character consistency, utilizing a Cloudflare Tunnel for secure, persistent reverse-proxying.

## **2\. System Architecture & Topology**

### **2.1 The "Bridge" Design**

The system "decouples the command center from the render farm" to prevent VRAM fragmentation and translation overhead on Apple Silicon.

* **Local UI (Mac):** Collects prompts, manages local storage (External SSD), and polls job status.  
* **Bridge URL:** A .trycloudflare.com endpoint generated by cloudflared on the remote node.  
* **Remote Node (Colab):** Executes heavy inference (Flux.1-schnell / img2vid) and exposes a job-based API.

### **2.2 Data Flow**

Mac UI → Bridge URL → FastAPI → Model Inference → Output Save → Mac Download

* *Optional:* Remote node saves to Google Drive (Direct-to-Drive) prior to local download.

## **3\. Canon Rules (Non-Negotiable)**

1. **Stateless-ish Remote:** Every output must be downloadable immediately to the local SSD. The remote node must handle session restarts without data loss or filename collisions.  
2. **Asynchronous Polling:** Video renders block the GPU but must not block the HTTP connection. Orchestrator submits, then polls /v1/jobs/{id}.  
3. **Character Lock:** Consistency is enforced via the "Character Bible Method"—a descriptor string prepended to every prompt.  
4. **Background Tunneling:** The Cloudflare tunnel must run in a background thread to avoid blocking the FastAPI server or the Colab notebook execution.

## **4\. Technical Specification: Remote Node (Colab)**

### **4.1 VRAM & Memory Strategy**

* **GPU:** NVIDIA T4 (16GB VRAM).  
* **Quantization:** Use NF4 (bitsandbytes) for Flux.1-schnell to ensure stability and multi-request longevity.  
* **Concurrency:** Global GPU Lock. Only one inference job runs at a time.  
* **Cleanup:** Explicit gc.collect() and torch.cuda.empty\_cache() after every job completion.

### **4.2 Notebook Cell Order**

1. **Dependencies:** fastapi, uvicorn, nest\_asyncio, diffusers, transformers, bitsandbytes, accelerate.  
2. **Drive Mount:** (Optional) /content/drive.  
3. **Tunnel Setup:** Download cloudflared binary; set executable permissions.  
4. **Server Engine:** FastAPI app, job queue, worker thread, and model loaders.  
5. **Execution:** Start Uvicorn and cloudflared in background threads.

## **5\. The API Contract (Bridge Protocol)**

All endpoints reside under /v1. Every request must include X-API-Key.

### **5.1 System Endpoints**

* GET /v1/health: Returns online status and GPU identification.  
* GET /v1/stats: Returns uptime and CUDA VRAM usage (VRAM Monitor).

### **5.2 Job Endpoints**

* **Job Object:** { job\_id, kind, status (queued|running|done|error), progress, result\_filename }  
* POST /v1/generate: (Text → Image)  
  * *Fields:* prompt, character\_lock, style\_preset, negative\_prompt, seed, width, height, steps, save\_to\_drive.  
* POST /v1/animate: (Image → Video)  
  * *Fields:* file (Multipart), prompt, fps, num\_frames, motion\_strength.  
* GET /v1/jobs/{job\_id}: Polls current status.  
* GET /v1/jobs/{job\_id}/download: Streams the resulting .png or .mp4.

## **6\. Technical Specification: Local Orchestrator (Mac)**

### **6.1 UI Structure (Gradio)**

* **Global Header:** Bridge URL, API Key, Local Storage Path (/Volumes/ExternalSSD/Starsilk\_Renders).  
* **Tab 1: Flux Forge:** Image generation, Prompt Matrixing, Character Bible injection, Seed Locking.  
* **Tab 2: Silk Motion:** Image-to-Video upload, motion parameters, MP4 preview.  
* **Tab 3: Starsilk Library:** Gallery view of local renders and file paths to external SSD.

### **6.2 Storage & Metadata**

* **Convention:** YYYYMMDD\_HHMMSS\_\_{kind}\_\_seed\_\_jobid\_\_slug.{ext}  
* **Metadata:** Every render generates a companion .json file containing the full prompt assembly, seed, and remote job stats.

## **7\. Starsilk Standards & Consistency**

### **7.1 Prompt Assembly Order**

Final Prompt \= \[CHARACTER\_LOCK\], \[STYLE\_PRESET\], \[SCENE\_PROMPT\]

### **7.2 The Character Bible (Example)**

"A cel-shaded small black-and-white dog, floppy ears down, human-like eyes, thick clean outlines, flat colors, consistent facial markings, no breed morphing."

### **7.3 Negative Prompt Master**

"blurry, distorted, low quality, gradient backgrounds, realistic textures, 3D render, over-detailed, inconsistent line weight."

## **8\. Mandatory Feature Set (Top 10\)**

| Feature | Implementation Detail |
| :---- | :---- |
| **Prompt Matrixing** | Multi-line input queue for batching variations. |
| **Character Lock** | Immutable trait injection (Starsilk lore enforcement). |
| **Seed Persistence** | UI toggle to lock seed for iterative refinements. |
| **Direct-to-Drive** | Boolean flag to mirror Colab outputs to Google Drive. |
| **Style Presets** | Dropdown for "Cel-shaded", "Flat Animation", "Glow". |
| **Batch Frame Extraction** | Local FFmpeg hook to split MP4s into PNG sequences. |
| **VRAM Monitor** | Real-time remote memory stats displayed in local UI. |
| **Negative Master** | Global kill-list prepended to every negative prompt field. |
| **Auto-Upscale (V1.1)** | Remote post-processing hook for high-res output. |
| **Workspace Integration** | Path detection for external storage mounts. |

## **9\. Verification Protocol**

1. **Tunnel Resilience:** Ensure FastAPI responds to health checks while the tunnel process is active in the background.  
2. **Event Loop Safety:** Verify nest\_asyncio is active to prevent Uvicorn/Notebook conflicts.  
3. **UI Resilience:** Polling must handle remote timeouts. If the Colab session dies, the UI must display "Remote Unreachable" without freezing.  
4. **Consistency Test:** Generate 5 images using the same Character Lock and Seed with different scene prompts. Traits must remain identical.
